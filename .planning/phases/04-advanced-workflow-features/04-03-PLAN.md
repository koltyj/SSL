    **Step 2: Add template CLI commands** to `SSLMatrixCLI` in `cli.py`:

    `do_template(self, arg)` — dispatcher for template subcommands. Parse `arg` to determine subcommand:
    - `template save [daw_path]` — Requires connected. Acquire `_lock`, capture state, release lock, call `save_template()`. Print saved filename.
    - `template list` — Call `list_templates()`. Print table: filename, title, saved_at. No connection required.
    - `template show <filename>` — Call `show_template()`. Pretty-print template contents (channels, layers, routing summary — not raw JSON). No connection required.
    - `template delete <filename>` — Call `delete_template()`. Confirm with prompt in REPL mode. No connection required.
    - `template load <filename> [--categories cat1,cat2]` — Requires connected. Load template, compute diff against current state under `_lock`, display diff grouped by category. In REPL mode: prompt "Apply [all/routing/channels/profiles/display/none]?". In one-shot mode: use --categories flag (default "all"). Build apply commands, send each with 50ms inter-packet delay (matching `request_sync()` pattern). Print full report: applied, skipped, unchanged.

    Thread safety for template load: acquire `_lock` to read current state for diff computation, copy needed fields, release lock, then display diff and prompt. After user confirms, acquire lock again to verify state hasn't changed, then send commands without lock (send_raw doesn't need lock). Follow the pattern from `do_status()`.

    **Step 3: Add split CLI command:**

    `do_split(self, arg)` — Parse arg:
    - `split <left_protocol> <right_protocol>` (e.g., `split HUI MCU`) — look up protocol names, find matching DAW layers, call `self.client.set_split([left_layers], [right_layers])`. Print assignment summary and guidance: "Left 8 faders: Press DAW Layer {N} button on console. Right 8 faders: Press DAW Layer {M} button on console."
    - `split status` — Print current split config or "No split configured"
    - `split clear` — Call `self.client.clear_split()`, print confirmation.

    **Step 4: Add health CLI command:**

    `do_health(self, arg)` — Requires connected. Under `_lock`, read: `desk.online`, `desk.heartbeat_age`, `desk.last_heartbeat`, `client._reconnecting`, `client._reconnect_attempts`. Print formatted health report:
    ```
    Connection: Online
    Heartbeat: 3.2s ago (healthy)
    Watchdog: Active
    Reconnects: 0
    ```
    If offline: show "Connection: Offline" with reconnect status.

    **Step 5: Add argparse subcommands** in the `main()` function for one-shot mode:
    - `template save`, `template list`, `template show`, `template delete`, `template load` (with `--categories` flag)
    - `split` (with subcommand)
    - `health`

    Follow existing argparse pattern from `main()`.
