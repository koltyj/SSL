  <files>ssl-matrix-client/client.py, tests/test_watchdog.py</files>
  <behavior>
    - TestWatchdog: When heartbeat_age exceeds HEARTBEAT_TIMEOUT (35s) and desk is online, _trigger_reconnect is called
    - TestWatchdog: When heartbeat_age is below threshold, no reconnect triggered
    - TestWatchdog: When desk is offline, watchdog does not trigger reconnect (already disconnected)
    - TestReconnectGuard: When _reconnecting is True, watchdog skips reconnect even if heartbeat is stale
    - TestReconnectGuard: _reconnecting is set True in _trigger_reconnect, cleared when GET_DESK_REPLY received (desk goes online)
    - TestReconnectAttempts: After MAX_RECONNECT_ATTEMPTS (10) without response, watchdog stops retrying and logs error
    - TestReconnectSync: After successful reconnect (desk.online becomes True), request_sync() is called automatically
  </behavior>
  <action>
    Extend `SSLMatrixClient` in `ssl-matrix-client/client.py`:

    1. Add constants at module level:
       ```python
       HEARTBEAT_TIMEOUT = 35.0    # Console sends heartbeat ~every 30s
       WATCHDOG_INTERVAL = 5.0     # Check every 5s
       RECONNECT_DELAY = 5.0       # Between reconnect attempts
       MAX_RECONNECT_ATTEMPTS = 10
       ```

    2. Add instance variables in `__init__`:
       - `self._watchdog_thread = None`
       - `self._reconnecting = False`
       - `self._reconnect_attempts = 0`

    3. Add `_watchdog_loop(self)` method:
       - Runs while `self._running`
       - Sleeps `WATCHDOG_INTERVAL` seconds between checks
       - Under `self._lock`, reads `desk.online`, `desk.heartbeat_age`, `self._reconnecting`
       - If online AND heartbeat_age > HEARTBEAT_TIMEOUT AND NOT _reconnecting: call `_trigger_reconnect()`
       - If _reconnecting AND _reconnect_attempts >= MAX_RECONNECT_ATTEMPTS: log error, set _reconnecting = False, reset attempts

    4. Add `_trigger_reconnect(self)` method:
       - Set `_reconnecting = True` under lock
       - Set `state.desk.online = False` under lock
       - Increment `_reconnect_attempts`
       - Log warning with attempt count
       - Call `_send_get_desk()`

    5. Modify `_recv_loop` GET_DESK_REPLY handling: after `handle_get_desk_reply` sets `desk.online = True`, check if `_reconnecting` was True. If so, set `_reconnecting = False`, reset `_reconnect_attempts = 0`, log "Reconnected", and schedule `request_sync()` call. Use a simple approach: start a short-lived thread that calls `request_sync()` (since we're inside `_lock` in the dispatch section, we cannot call request_sync directly — it would deadlock trying to acquire the lock via `wait_online`). Alternative: set a flag `_needs_resync = True` and check it in the watchdog loop.

    Actually, simpler approach for re-sync: In `_recv_loop`, after releasing the lock from the handler dispatch, check a `_needs_resync` flag. If True, call `request_sync()` outside the lock. This avoids threading complexity.

    6. Modify `connect()`: start watchdog thread after recv thread.
    7. Modify `disconnect()`: stop and join watchdog thread.

    Scope note per research Open Question 1: This monitors SSL UDP heartbeat only, NOT ipMIDI. ipMIDI runs on a separate port/protocol. Document this in docstrings.

    For tests: Mock `time.time()` and `time.sleep()` to control timing. Mock `_send_get_desk` and `request_sync` to verify they're called. Use `unittest.mock.patch` on the client instance methods. Do not require a real socket.
