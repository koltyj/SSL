Create `ssl-matrix-client/tui_widgets.py`:

1. **ChannelStrip(Static)** — A single channel strip widget (fixed width ~10-12 chars):
   - Reactive attributes: `channel_name: reactive[str]`, `channel_num: reactive[int]`, `daw_protocol: reactive[str]`, `auto_mode: reactive[str]`, `insert_routing: reactive[str]`, `highlighted: reactive[bool]`
   - `__init__(self, num: int)`: Set `channel_num = num`, set CSS id to `f"ch-{num}"`
   - `render()`: Return Rich markup with vertical layout inside the strip:
     - Top: channel number (dim)
     - Middle: channel name (bold, primary color) — this is the "scribble strip" label
     - Below name: insert routing info (dim, shows chain name or insert device numbers, e.g. "Ins: 3,7" or chain name if assigned). If no inserts, show "Ins: --"
     - Bottom: DAW protocol + automation mode (dim, smaller)
   - `watch_channel_name(self, old, new)`: If old is not empty and differs from new, call `self.flash_highlight()`; always call `self._refresh_content()` (or just `self.update(self.render())`)
   - `watch_daw_protocol`, `watch_auto_mode`, `watch_insert_routing`: Same flash pattern
   - `watch_highlighted(self, value)`: Add/remove `"highlighted"` CSS class
   - `flash_highlight(self)`: Set `self.highlighted = True`, then `self.set_timer(1.5, lambda: setattr(self, 'highlighted', False))`
   - `update_from_snapshot(self, ch_tuple, daw_layer_tuple, auto_mode_int, insert_info)`:
     Set reactive attrs from snapshot data. `ch_tuple` is `(num, name)`, `daw_layer_tuple` is `(num, protocol_int, profile_name)`. Map protocol int to name: `{0: "", 1: "HUI", 2: "MCU", 3: "CC"}`. Map auto_mode: `{0: "Legacy", 1: "Delta"}`.
     `insert_info` is `(channel, chain_name, [insert_nums], has_stereo)` or `None`. Set `self.insert_routing` from this:
     - If `insert_info` and `insert_info[1]` (chain_name): show chain_name
     - Elif `insert_info` and `insert_info[2]` (insert list non-empty): show `"Ins: {','.join(str(n) for n in insert_info[2])}"`
     - Else: `"Ins: --"`

2. **ChannelView(HorizontalScroll)** — Container for 16 channel strips:
   - `compose()`: Yield 16 `ChannelStrip(i)` for `i in range(1, 17)`
   - `update_from(self, snapshot)`: For each channel strip, find its data in `snapshot["channels"]` (list of `(num, name)` tuples) and the corresponding `daw_layer` (channels 1-16 map to the first active DAW layer), then call `strip.update_from_snapshot(...)`.
     - DAW layer mapping: The console has 4 layers. For simplicity, show the protocol of daw_layer[0] for all channels (the TUI shows which layer is currently controlling the surface). The user can see all 4 layers in the Settings tab later.
     - Insert routing: Build a dict from `snapshot["channel_inserts"]` keyed by channel number. For each strip, look up the channel's insert info (or pass `None` if not found).

3. **DisconnectOverlay(ModalScreen)** — Full-screen modal:
   - `DEFAULT_CSS`: Semi-transparent background (`background: $background 70%`), centered content box with `$error` border
   - `__init__(self, attempt: int = 0)`: Store attempt count
   - `compose()`: Yield a `Vertical` container with:
     - `Label("[bold]DISCONNECTED[/]")` styled with error color
     - `Label(f"Reconnecting... attempt {self.attempt}")`
     - `Label("Press Ctrl+Q to quit")`
   - `update_attempt(self, attempt)`: Update the attempt label text

2. **Wire StateUpdated to ChannelView**: In `on_ssl_app_state_updated(self, msg)`, after updating the status bar, also call:
   ```python
   channel_view = self.query_one(ChannelView, default=None)
   if channel_view is not None:
       channel_view.update_from(msg.snapshot)
   ```
   Use `default=None` guard because the widget may not be mounted yet during initial compose.

3. **Handle ConsoleOffline**: Add `on_ssl_app_console_offline(self, msg)` handler:
   - Push `DisconnectOverlay(attempt=msg.attempt)` as a modal screen
   - Store reference: `self._disconnect_overlay = ...` so we can update attempt count on subsequent offline messages
   - If overlay already pushed (check `self._disconnect_overlay`), update the attempt label instead of pushing again

4. **Handle ConsoleOnline**: Add `on_ssl_app_console_online(self, msg)` handler:
   - Pop the disconnect overlay: `self.pop_screen()` (only if overlay is currently showing)
   - Clear `self._disconnect_overlay = None`

5. **Handle subsequent offline attempts while overlay is showing**: The watchdog sends multiple `ConsoleOffline` messages with incrementing attempt counts. The handler must update the existing overlay's attempt label rather than pushing multiple overlays. Pattern:
   ```python
   def on_ssl_app_console_offline(self, msg):
       if self._disconnect_overlay is not None:
           # Update existing overlay
           self._disconnect_overlay.attempt = msg.attempt
           # Update label text
           ...
       else:
           overlay = DisconnectOverlay(attempt=msg.attempt)
           self._disconnect_overlay = overlay
           self.push_screen(overlay)
   ```

6. **Initialize `_disconnect_overlay = None`** in `__init__`.
