**Create `ssl-matrix-client/tui_commands.py`:**

1. **ConsoleCmdProvider(Provider)** subclass:
   - Define a list of command entries, each with: `name` (what user types), `help` (description), `callback_name` (method name on SSLApp to call).
   - Commands to include (a practical subset — not all 30 REPL commands):
     - `"sync"` / `"Refresh console state"` — calls `request_sync` in a worker thread
     - `"channels"` / `"Show channel names"` — switches to channels tab
     - `"routing"` / `"Show insert routing"` — switches to routing tab
     - `"templates"` / `"Show session templates"` — switches to templates tab
     - `"settings"` / `"Show console settings"` — switches to settings tab
     - `"template save <name>"` / `"Save session template"` — future: prompt for name
     - `"template load <name>"` / `"Load session template"` — future: prompt for name
     - `"split"` / `"Show split board status"` — displays current split config
     - `"auto legacy"` / `"Set automation mode to Legacy"` — calls `set_auto_mode(0)`
     - `"auto delta"` / `"Set automation mode to Delta"` — calls `set_auto_mode(1)`
     - `"motors on"` / `"Enable motors"` — calls `set_motors_off(0)`
     - `"motors off"` / `"Disable motors"` — calls `set_motors_off(1)`
     - `"quit"` / `"Exit TUI"` — calls `app.exit()`

   - `async def search(self, query: str) -> Hits`: Use `self.matcher(query)` for fuzzy matching against command names. Yield `Hit(score, matcher.highlight(name), callback, help=description)`.

   - For callbacks that need the client, access it via `self.app.client`. For callbacks that mutate state via UDP, run them in a worker thread to avoid blocking the async loop: `self.app.run_worker(lambda: self.app.client.method(), thread=True)`.

   - For tab-switching callbacks, call `self.app.action_show_tab("tab_id")` directly (async-safe).

**Create `ssl-matrix-client/tui_views.py`:**

2. **RoutingView(Static)** — Read-only view of insert routing:
   - `update_from(self, snapshot)`: For now, display a formatted table of insert devices and channel assignments from the snapshot. The snapshot from Plan 01 does not include routing data, so this view will need the snapshot to be extended OR read directly from `self.app.client` under lock.
   - Approach: On tab activation (or on `StateUpdated`), read from `self.app.client` under lock:
     ```python
     with self.app.client._lock:
         devices = list(self.app.client.state.devices)
         channel_inserts = list(self.app.client.state.channel_inserts)
     ```
   - Render as formatted text: device number, name, assigned status. Then channel insert assignments.
   - Use Rich markup for formatting (tables/columns).

3. **TemplatesView(Static)** — Show saved templates:
   - On mount or refresh, call `templates.list_templates()` to get template file paths.
   - Render as a list: template name, file date/size.
   - This is a static read — no live updates needed.

4. **SettingsView(Static)** — Console configuration summary:
   - `update_from(self, snapshot)`: Display:
     - Console name and firmware version (from `self.app.client.state.desk` under lock)
     - DAW layers: layer number, protocol, profile name
     - Automation mode (Legacy/Delta)
     - Motors status, MDAC meters
     - Split board config if active
   - Read from client state under lock on each `StateUpdated`.

Update `ssl-matrix-client/ssl_theme.tcss`:
