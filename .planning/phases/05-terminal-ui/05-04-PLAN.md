  <action>
Present the verification checklist to the user and wait for confirmation. This is a human-verify checkpoint — no automated work to perform.

What was built: Complete terminal UI for SSL Matrix console monitoring and control: channel strips view with live updates, tabbed navigation, command palette, status bar with health dot, disconnect overlay, and SSL-inspired theme.

How to verify:
1. Ensure the SSL Matrix console is powered on and connected via Ethernet.

2. Launch the TUI:
   ```
   python3 -m ssl-matrix-client tui
   ```

3. **Connection check:** Status bar at the bottom should show a green health dot and the active project name. If red dot or "DISCONNECTED" overlay appears, check network connectivity.

4. **Channel strips (Tab 1):** You should see 16 channel strips in a horizontal row. Each strip shows the channel number, name (from the console's scribble strips), DAW protocol, and automation mode. The layout should visually evoke the physical console surface.

5. **Tab switching:** Press `2` for Routing, `3` for Templates, `4` for Settings, `1` to return to Channels. Each tab should show relevant content (device routing, saved templates, DAW layer config).

6. **Command palette:** Press `:` or `/` to open the command palette. Type "sync" — it should appear in the fuzzy search results. Select it to trigger a full state refresh. Try typing "auto" to see automation mode commands.

7. **Live updates (if possible):** Change a channel name on the console hardware or via a second terminal running the REPL (`python3 -m ssl-matrix-client` then `rename 1 TEST`). The TUI should update the channel strip within 1-2 seconds, with a brief amber flash on the changed strip.

8. **Theme check:** The overall look should be dark with green and amber accents — inspired by the SSL console hardware aesthetic. Not a generic terminal app.

9. **Exit:** Press `q` or `Ctrl+C` to quit cleanly.

Resume signal: Type "approved" if the TUI works correctly, or describe any issues (visual, functional, connection) to fix.
