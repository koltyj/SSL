<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Connect MatrixRemote and configure Pro Tools HUI</name>
  <action>
Claude provides instructions; user performs physical and GUI steps.

**Step 1 — Connect MatrixRemote to the console:**
1. Ensure the SSL Matrix is powered on and the Ethernet cable is connected
2. Launch MatrixRemote
3. MatrixRemote should auto-detect the console on the local network. Look for "Matrix Connected - Matrix SN XXXX" in the status bar
4. If MatrixRemote does NOT detect the console:
   - Verify the Matrix IP is reachable: `ping <matrix-ip>` from Terminal
   - Check that MatrixRemote has Local Network permission (System Settings > Privacy & Security > Local Network)
   - Try quitting and relaunching MatrixRemote
   - If MatrixRemote shows Java errors or rendering issues, verify Java version with `java -version`
5. Once connected, note the firmware version shown in MatrixRemote's connection info (if visible)

**Step 2 — Configure MatrixRemote for HUI/DAW control:**
1. In MatrixRemote, navigate to the DAW Control or Profile settings
2. Select or create a HUI profile
3. Assign ipMIDI ports to HUI banks. The Matrix uses multiple ipMIDI ports for multiple HUI zones (each HUI zone controls 8 faders):
   - HUI Zone 1: ipMIDI Port 1 (faders 1-8)
   - HUI Zone 2: ipMIDI Port 2 (faders 9-16)
   - Additional zones as needed for the console's fader count
4. Save the profile

**Step 3 — Verify MIDI traffic with MIDI Monitor:**
1. Open MIDI Monitor (Snoize)
2. Set it to listen on all ipMIDI sources
3. Move a fader on the Matrix — MIDI Monitor should show incoming MIDI messages (HUI fader data)
4. If MIDI Monitor shows NO traffic from the Matrix:
   - Multicast routing may be wrong — revisit Plan 01 Task 2 Step 5
   - Local Network permission may have been lost — check System Settings
   - ipMIDI port configuration may be mismatched with MatrixRemote's port assignment
5. Take a screenshot or describe the MIDI traffic seen

**Step 4 — Configure Pro Tools for HUI control surface:**
1. **CRITICAL:** Before configuring HUI, disable EUCON. Go to Pro Tools > Setup > Peripherals > Ethernet Controllers and confirm EuControl is NOT active. If it is active, disable it and restart Pro Tools. (Leaving EUCON on causes a metering bug on HUI surfaces.)
2. Go to Pro Tools > Setup > Peripherals > MIDI Controllers (or HUI tab, depending on Pro Tools version)
3. Add a HUI controller:
   - Receive From: ipMIDI Port 1
   - Send To: ipMIDI Port 1
4. Add a second HUI controller for Port 2 if using two HUI zones
5. Click OK to apply

**Step 5 — Create or open a test session:**
1. Create a new Pro Tools session with at least 4 audio tracks
2. On tracks 1-4, write simple fader automation (e.g., a volume ramp from -inf to 0 dB over 8 bars)
3. Set all tracks to automation mode "Read"
4. Save the session

<task type="checkpoint:human-verify" gate="blocking">
  <name>Task 2: Verify the four mandatory DAW behaviors</name>
  <action>
The complete protocol stack (SSL Matrix → Ethernet → ipMIDI → CoreMIDI → Pro Tools HUI) was configured in Task 1. The user now runs through all four mandatory verification behaviors. Each must PASS for Phase 1 to be complete.

**Behavior 1 — Automation playback moves physical faders:**
1. In Pro Tools, press Play on the transport bar (not the console — use mouse or keyboard first)
2. Watch the Matrix console faders on channels 1-4
3. PASS: Faders physically move in sync with the automation curves written in the test session
4. FAIL: Faders do not move at all, or move erratically / with significant lag

**Behavior 2 — Transport buttons control Pro Tools:**
1. Stop playback using the Matrix's transport section (Stop button on the console)
2. Press Play on the Matrix console
3. Press Record on the Matrix console (with a record-enabled track)
4. Press Fast Forward and Rewind on the Matrix console
5. PASS: Each transport action is reflected in Pro Tools (play cursor moves, recording starts, etc.)
6. FAIL: Transport buttons have no effect in Pro Tools

**Behavior 3 — Fader touch writes automation:**
1. Set one track to automation mode "Touch" in Pro Tools
2. Start playback
3. Touch and move the corresponding fader on the Matrix console
4. Release the fader
5. Stop playback and examine the automation lane in Pro Tools
6. PASS: The automation lane shows a new write corresponding to your fader movement, and the fader returns to the previously written automation after release
7. FAIL: No automation is written, or Pro Tools does not recognize fader touch

**Behavior 4 — Track names on scribble strips:**
1. Look at the Matrix console's scribble strip displays (small text displays above or below each fader)
2. Compare with Pro Tools track names for channels 1-4
3. PASS: Matrix scribble strips show the Pro Tools track names (may be truncated to fit the display width)
4. FAIL: Scribble strips are blank, show "---", or show incorrect names

**If any behavior FAILS:** Do not give up. Report the exact failure to Claude with which behavior failed, what was observed, and any error messages. Claude will troubleshoot using the research document's pitfall guides and fallback decision tree.
