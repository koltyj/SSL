<task type="checkpoint:human-action" gate="blocking">
  <name>Task 1: Identify firmware, install software, and grant permissions</name>
  <action>
This task requires the user to perform physical and GUI actions that Claude cannot automate. Claude provides the checklist; user executes.

**Step 1 — Identify Matrix firmware version:**
1. Power on the SSL Matrix console
2. Watch the console's built-in alphanumeric display during bootup — it shows the firmware version string (e.g., "V2.0/6")
3. Report the firmware version to Claude

**Step 2 — Install Java runtime (if needed):**
1. Open Terminal and run: `java -version`
2. If Java is not installed or is below version 11: download and install Adoptium Temurin JDK 21 LTS from https://adoptium.net/ (macOS ARM64 installer)
3. After install, confirm: `java -version` shows 21.x

**Step 3 — Download and install MatrixRemote:**
1. Go to SSL Support for the ORIGINAL Matrix (NOT Matrix 2): https://support.solidstatelogic.com/hc/en-gb/articles/4407299066385-Matrix-Documents
2. Download the MatrixRemote version matching the firmware identified in Step 1
3. If the page redirects to Matrix 2 software, do NOT install it — contact SSL support for the legacy download link
4. Install MatrixRemote (drag to /Applications or follow installer)
5. Launch MatrixRemote once to confirm it opens (it will not connect to the console yet — that is Plan 02)

**Step 4 — Download and install ipMIDI 2.0:**
1. Download ipMIDI from https://nerds.de/en/download.html (macOS version, 2.0)
2. Mount the DMG and run the installer
3. Reboot the Mac after installation (required for the driver to register with CoreMIDI)
4. After reboot, open System Settings and locate the ipMIDI preference pane
5. Configure: set port count to 9, disable loopback

**Step 5 — Install MIDI Monitor:**
1. Download from https://www.snoize.com/MIDIMonitor/
2. Drag to /Applications
3. Launch once to confirm it opens

**Step 6 — Grant Local Network permissions on Tahoe:**
1. After launching ipMIDI for the first time, macOS will prompt for Local Network permission — grant it
2. If MatrixRemote prompts for Local Network permission — grant it
3. Go to System Settings > Privacy & Security > Local Network and verify both ipMIDI and MatrixRemote have permission enabled
4. **CRITICAL (Tahoe bug):** Reboot the Mac once more
5. After reboot, re-check System Settings > Privacy & Security > Local Network — confirm ipMIDI and MatrixRemote entries still appear and are enabled
6. If either entry disappeared after reboot: open Terminal and run `tccutil reset LocalNetwork`, then relaunch the affected app and re-grant permission. Reboot and verify again.

<task type="checkpoint:human-action" gate="blocking">
  <name>Task 2: Map network topology and configure multicast routing</name>
  <action>
Claude will provide terminal commands; user runs them and shares output. Claude analyzes results.

**Step 1 — Map physical connections:**
1. Physically trace the Ethernet cable from the SSL Matrix console rear panel
2. Identify every device in the chain: Matrix → [cable] → [what device?] → [cable?] → Mac
3. Specifically determine: is the Matrix connected to the USB hub/dock's built-in Ethernet? Or directly to a switch? Or directly to the Thunderbolt Ethernet adapter?
4. Report the physical chain to Claude

**Step 2 — Identify macOS network interfaces:**
Run in Terminal and share the output:
```bash
networksetup -listallhardwareports
```
This lists all interfaces (Wi-Fi, Thunderbolt Ethernet, USB Ethernet, etc.) with their device names (en0, en1, etc.).

**Step 3 — Determine which interface reaches the Matrix:**
1. Find the Matrix's IP address from its console display menu (likely 192.168.1.x — check under Network or System settings on the console itself)
2. Report the Matrix's IP address
3. Run: `ping -c 3 <matrix-ip>` and share output — if it responds, the Mac can reach the Matrix
4. If ping fails: the Matrix and Mac may not be on the same subnet. Claude will help troubleshoot.

**Step 4 — Check current multicast routing:**
Run in Terminal:
```bash
netstat -rn | grep 225
```
If any route for 225.x.x.x already exists, share the output. If empty, no multicast route is configured yet.

**Step 5 — Add static multicast route:**
Claude will determine the correct interface name from Steps 1-3. The user then runs:
```bash
# Replace <interface-name> with the actual interface (e.g., en3, en5)
sudo route -n add -net 225.0.0.37 -interface <interface-name>
```
Then verify:
```bash
netstat -rn | grep 225
```
This route is ephemeral (lost on reboot). For persistence, Claude will provide the `networksetup -setadditionalroutes` command using the correct service name from Step 2.

**Step 6 — Verify ipMIDI ports in Audio MIDI Setup:**
1. Open Audio MIDI Setup (Applications > Utilities)
2. Go to Window > Show MIDI Studio (or MIDI Devices)
3. Look for ipMIDI ports — there should be up to 9 ports listed
4. Take a screenshot or describe what appears
5. Also run in Terminal: `system_profiler SPMIDIDataType` and share output
  </action>
  <verify>
User reports:
1. Physical network path from Matrix to Mac is documented
2. Matrix IP address is known and Mac can ping it
3. `netstat -rn | grep 225` shows a route for 225.0.0.37 via the correct interface
4. Audio MIDI Setup > MIDI Studio shows ipMIDI ports
5. `system_profiler SPMIDIDataType` lists ipMIDI as a MIDI source
  </verify>
  <done>Network topology is mapped, the Mac can reach the Matrix over Ethernet, multicast routing for 225.0.0.37 is directed to the correct interface, and ipMIDI ports are visible as CoreMIDI sources in Audio MIDI Setup.</done>
