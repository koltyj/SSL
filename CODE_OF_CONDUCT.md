# Code Of Conduct

## Our Standard

This project expects respectful, direct, and technically grounded collaboration.

Examples of expected behavior:

- Be constructive in reviews and issue discussions.
- Critique code and ideas, not people.
- Share context when reporting bugs or regressions.
- Respect different experience levels and studio setups.

Examples of unacceptable behavior:

- Harassment, intimidation, or personal attacks
- Discriminatory language or behavior
- Doxxing or sharing private information without consent
- Repeated bad-faith arguments that derail technical work

## Scope

This applies to project spaces, including issues, pull requests, discussions, and any linked communication channels used for project collaboration.

## Enforcement

Project maintainers may edit, remove, or reject content that violates this code of conduct. Repeated or severe violations may lead to temporary or permanent exclusion from project spaces.

## Reporting

If you experience or witness unacceptable behavior, contact the maintainers through GitHub. Include links or screenshots when possible so the report can be reviewed quickly.
